# M4 – QA, 배포, 최소 법적 요건  
> 통합 QA · 프로덕션 배포 · 개인정보 처리방침 · 2차 백로그

---

## M4-WP1. 시나리오 기반 통합 QA

**사용자 가치**  
실제 사용 상황에서 전체 플로우가 끊김 없이 동작함을 검증한다.

**Scope**  
- 대표 시나리오 테스트
  - 보호자: 가입 → 케이스 생성 → 계약 동의 → 기간 변경 → 지급 정보 → PDF 생성
  - 간병인: 링크 수신 → 동의 → 여러 날짜 일지 작성
- 엣지 시나리오
  - 조기 종료 후 일지 동작
  - 연장 후 추가 일지 작성
  - 일부 일지 누락 상태에서 PDF 생성 시도
- 발견된 버그 목록화 및 우선순위별 처리

**완료 기준**  
- 치명적 버그(플로우 진행 불가)가 제거된 상태
- 주요 테스트 케이스가 문서/체크리스트로 남아 있음

## M4-WP2. 프로덕션 배포 환경 세팅

**사용자 가치**  
실제 서비스 URL에서 보호자·간병인 모두 사용할 수 있다.

**Scope**  
- Vercel 프로덕션 프로젝트 생성 및 연결
- Supabase 프로덕션 DB 연결
- dev/prod 환경변수 정리
- 배포 후 기본 기능 헬스 체크
  - 로그인, 케이스 생성, 링크 접속, PDF 생성 등

**완료 기준**  
- 프로덕션 URL에서 전체 주요 플로우가 정상 동작
- 개발용 환경과 프로덕션 환경 설정이 명확히 구분

---

## M4-WP3. 개인정보 처리방침 페이지

**사용자 가치**  
민감 정보 처리에 대한 최소한의 신뢰와 법적 방어선을 갖춘다.

**Scope**  
- `/privacy` 페이지 추가
  - 수집 항목(이름, 연락처, 주민번호, 계좌 등)
  - 이용 목적(간병 서류 생성, 보험 청구용 등)
  - 보관 기간 및 파기 방침(초기 버전이라도 명시)
- 앱 내에서 해당 페이지 링크 노출
  - 로그인 화면/푸터 등

**완료 기준**  
- 사용자가 언제든지 개인정보 처리방침을 확인 가능
- 현재 서비스 범위에 맞는 내용이 포함

## M4-WP4. 2차 Milestone 백로그 정리

**사용자 가치**  
MVP 이후 발전 방향이 명확해진다.

**Scope**  
- 개발/QA 과정에서 나온 “다음 단계” 후보 정리
  - 전자 서명(보호자·간병인)
  - 주민번호 암호화/부분 수집
  - 카카오 알림톡/자동 알림
  - Supabase RLS, 세션 최적화, 레이트 리밋 등
- 중요도/난이도 기반 우선순위 부여

**완료 기준**  
- 2차 이후 수행할 작업 리스트가 정리된 문서로 존재
- 1차 릴리즈 이후의 로드맵 초안 확보
