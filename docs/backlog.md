# 2차 Milestone 백로그 (MVP 이후)

본 문서는 1차 MVP 개발 및 QA 과정에서 도출된 개선 필요 항목과 2차 Milestone에서 수행할 작업 목록을 정리한 것입니다.

## 1. 우선순위 높음 (High Priority)

### 1.1 전자 서명 도입
- **현재**: 단순 체크박스 동의
- **개선**: 간편 전자 서명(Canvas 서명판) 또는 외부 서명 서비스 연동 (ex. 모두싸인 API)
- **목적**: 계약의 법적 효력 강화 및 신뢰도 향상

### 1.2 개인정보 암호화 및 보안 강화
- **현재**: 주민등록번호 평문 저장 (DB 보안에 의존)
- **개선**: 
  - 주민등록번호 등 민감 정보의 DB 레벨 암호화 (pwsodium 등 활용)
  - 수집 최소화 (주민번호 뒷자리는 마스킹 처리 후 저장 등)
- **목적**: 개인정보보호법 준수 및 유출 사고 예방

### 1.3 알림 서비스 (Notification)
- **현재**: 사용자가 직접 접속하여 확인해야 함
- **개선**: 
  - 카카오 알림톡 연동 (계약 동의 요청, 일지 작성 완료 알림)
  - 이메일 알림 (중요 계약서 발송)
- **목적**: 서비스 사용성 및 반응 속도 개선

## 2. 우선순위 중간 (Medium Priority)

### 2.1 Supabase RLS 및 정책 최적화
- **현재**: 기본적인 `auth.uid()` 기반 RLS 적용
- **개선**: 
  - `case_tokens` 기반의 간병인 접근 권한을 RLS 정책(Function)으로 엄격하게 제어
  - Service Role 사용 최소화 및 권한 분리
- **목적**: 보안 홀 제거 및 안정성 확보

### 2.2 간병일지 고도화
- **현재**: 텍스트 기반 메모 및 단순 체크박스
- **개선**: 
  - 사진 첨부 기능 (식사 사진, 환자 상태 사진)
  - 활력 징후(혈압, 체온) 입력 필드 및 그래프 시각화
- **목적**: 보호자에게 더 풍부한 정보 제공

### 2.3 지급 관리 고도화
- **현재**: 단순 기록용
- **개선**: 
  - 실제 결제 연동 (PG사 연동)
  - 정산 내역서 자동 발송
- **목적**: 플랫폼 내 완결성 확보

## 3. 우선순위 낮음 / 보류 (Low Priority)

### 3.1 다국어 지원
- **개선**: 외국인 간병인을 위한 다국어(중국어, 베트남어 등) 인터페이스 제공

### 3.2 관리자 대시보드
- **개선**: 전체 케이스 현황, 통계, CS 처리를 위한 어드민 페이지 구축

---

## 4. 기술 부채 (Technical Debt)
- **테스트 커버리지 확보**: 핵심 비즈니스 로직(계산, 상태 변경)에 대한 Unit Test 추가
- **컴포넌트 리팩토링**: 중복된 UI 코드(버튼, 폼 등)를 공통 컴포넌트로 분리
- **에러 모니터링**: Sentry 등 에러 트래킹 도구 연동
